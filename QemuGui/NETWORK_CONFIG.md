# Сетевые настройки QEMU Launcher

## Режимы сети

### 1. User (Slirp) - рекомендуется
- **Описание**: Простой режим без привилегий, работает на всех платформах
- **Доступ к интернету**: Есть (исходящие соединения)
- **Доступ из хоста**: Ограничен (только через пробросы портов)
- **Требует администратора**: Нет
- **Параметры QEMU**: 
  ```
  -netdev user,id=net0
  -device e1000,netdev=net0,mac=52:54:00:12:34:56
  ```

### 2. Tap (Bridge) - для продвинутых
- **Описание**: Мост к физической сети, требует настройки на хосте
- **Доступ к интернету**: Полный (как реальный компьютер в сети)
- **Доступ из хоста**: Полный
- **Требует администратора**: Да
- **Параметры QEMU**: 
  ```
  -netdev tap,id=net0,ifname=tap0,script=no,downscript=no
  -device e1000,netdev=net0,mac=52:54:00:12:34:56
  ```
- **Требует предварительной настройки на Windows**:
  - Установить OpenVPN TAP adapter или использовать VirtualBox TAP
  - Создать виртуальный адаптер с именем `tap0`

### 3. None
- **Описание**: Сеть отключена
- **Доступ к интернету**: Нет
- **Требует администратора**: Нет

## MAC адреса

- **По умолчанию**: `52:54:00:12:34:56`
- **Генератор**: Нажмите кнопку "Generate" для создания случайного MAC
- **Формат**: `XX:XX:XX:XX:XX:XX` где X - шестнадцатеричная цифра
- **Первые три октета** `52:54:00` зарезервированы для QEMU/KVM

## Типы сетевых адаптеров

В текущей версии используется **e1000** (эмуляция Intel 1000 MBit Ethernet):
- Хорошая совместимость с гостевыми ОС
- Windows (XP и выше) имеет встроенные драйверы
- Linux поддерживает с ядром 2.6+

Альтернативные адаптеры:
- `virtio-net` - быстрее, требует драйверов VirtIO (может быть добавлено позже)
- `pcnet` - старый AMD-совместимый адаптер

## Пример использования

1. Выберите **User (Slirp)** для первого запуска
2. Оставьте MAC адрес по умолчанию или нажмите "Generate"
3. Запустите виртуальную машину
4. Внутри ОС настройте сеть (DHCP или статический IP)
5. Проверьте подключение (ping, браузер и т.д.)

## Решение проблем

### Нет интернета в User режиме
- Проверьте настройки брандмауэра гостевой ОС
- Попробуйте явно указать DNS: `8.8.8.8` или `1.1.1.1`

### TAP не работает
- Убедитесь, что адаптер `tap0` создан
- Проверьте права администратора при запуске QEMU
- На Windows может потребоваться установка сторонних инструментов

### MAC адрес конфликты
- Если несколько ВМ используют одинаковые MAC, используйте генератор
- Каждая ВМ должна иметь уникальный MAC в одной сети
